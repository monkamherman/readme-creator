Le système de livraison de repas multi-vendeurs Enatega est parfait pour les clients cherchant à déployer une plateforme de commande de nourriture prête à l'emploi et facile à utiliser pour leur propre entreprise de livraison de repas et de logistique. Tout comme foodpanda et ubereats, notre système de livraison de repas peut intégrer plusieurs restaurants ainsi que des restaurants opérant dans plusieurs lieux. Avec l'accès au panneau d'administration et des applications séparées pour les clients et les coursiers, vous pouvez utiliser cette solution pour créer votre propre système de commande de nourriture en ligne et de gestion des commandes/livraisons instantanément.

Notre solution est open source mais le backend et l'API sont propriétaires, et peuvent être obtenus via une licence payante.

## :question: Ce qui est inclus : <a id="heading-1"></a>

Notre système de livraison de repas comprend également les éléments suivants :

- Enatega Multi vendeur Application Client
- Enatega Multi vendeur Application Coursier
- Enatega Multi vendeur Application Restaurant
- Site Web de Commande de Nourriture Client
- Tableau de Bord Web Administrateur
- Serveur d'Interface de Programmation d'Application (API)
- Tableau de Bord Analytique avec Expo Amplitude
- Rapport de crash d'erreur avec Sentry

## :fire: Fonctionnalités : <a id="heading-2"></a>

- Authentification utilisant Google, Apple et Facebook
- Fonctionnalité de différentes sections pour promouvoir les restaurants
- Notifications Push et E-mails aux Utilisateurs pour la création de compte et les changements de statut de commande
- Suivi en temps réel du Coursier et option de chat avec le Coursier
- Vérification par E-mail et Numéro de téléphone
- Restaurants basés sur la localisation affichés sur la Carte et l'Écran d'Accueil
- Support Multi-Langue et différents thèmes
- Fonctionnalités de Note et Avis pour la commande
- Les détails des restaurants incluent les notes et avis, les horaires d'ouverture et de fermeture, les délais de livraison, le menu et les articles du restaurant, l'emplacement du restaurant, la commande minimum
- Intégration de Paiement pour PayPal et Stripe
- Historique des commandes précédentes et ajout de restaurants favoris
- Ajout d'adresse avec suggestions Google Places et intégration Maps
- Analytique et rapport d'Erreur avec Amplitude et Sentry
- Options pour ajouter différentes variations d'articles alimentaires et ajouter des notes au restaurant
- Option de retrait et livraison avec différents horaires

## :computer: Guide de Configuration du Projet <a id="heading-15"></a>

Cette section fournit des instructions détaillées pour configurer et exécuter chaque composant de la Solution de Livraison de Repas Multi-Vendeurs Enatega.

### Tableau de Bord Admin Enatega (Next.js)

Le tableau de bord admin vous permet de gérer les restaurants, les commandes, les coursiers, et plus encore.

```bash
# Naviguer vers le répertoire du tableau de bord admin
cd enatega-multivendor-admin

# Installer les dépendances
npm install

# Démarrer le serveur de développement
npm run dev
```

Après avoir exécuté ces commandes, ouvrez votre navigateur et naviguez vers [http://localhost:3000](http://localhost:3000) pour accéder au tableau de bord admin. Vous pouvez aussi faire CTRL+clic sur le lien localhost qui apparaît dans votre terminal.

### Web Client Enatega (React.js)

L'application web client permet aux utilisateurs de parcourir les restaurants et passer des commandes via un navigateur web.

```bash
# Naviguer vers le répertoire web client
cd enatega-multivendor-web

# Installer les dépendances
npm install

# Démarrer le serveur de développement
npm start
```

Après avoir exécuté ces commandes, l'application sera disponible à [http://localhost:3000](http://localhost:3000) dans votre navigateur web.

### App Client Enatega (React Native)

L'application mobile client permet aux utilisateurs de parcourir les restaurants et passer des commandes sur leurs appareils mobiles.

```bash
# Naviguer vers le répertoire app client
cd enatega-multivendor-app

# Installer les dépendances
npm install

# Démarrer le serveur de développement Expo
npx expo start -c
# OU
npm start -c
```

### Construire les Versions de Développement

Pour toutes les applications mobiles (Client, Coursier et Restaurant), vous pouvez créer des builds de développement en utilisant EAS Build.

#### Configurer EAS Build

```bash
# Depuis le répertoire de l'app (client, coursier, ou restaurant)
eas build:configure
```

Sélectionnez votre plateforme souhaitée :
- android
- ios
- all

#### Build pour Android

```bash
eas build --platform android --profile development
```

Cela créera un fichier APK que vous pouvez installer directement sur votre appareil Android.

#### Build pour iOS

```bash
eas build --platform ios --profile development
```

Pour les builds simulateur iOS, modifiez le fichier `eas.json` pour inclure :

```json
"development": {
  "developmentClient": true,
  "distribution": "internal",
  "channel": "development",
  "ios": {
    "simulator": true
  },
  "android": {
    "buildType": "apk"
  }
}
```

Puis exécutez :

```bash
eas build --platform ios --profile development
```
